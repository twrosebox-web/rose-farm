## 5. 預約功能方向（P8）

- **第一版價值 = 把 FB／電話／LINE／網站來源匯進同一本帳**，不是客人自助付款。
- 資料放**獨立私有 Sheet**（限定共用）、**獨立 Apps Script 部署**，驗證比內容後台更嚴（存個資、POST-only、不公開讀取）。
- 30 筆/月：做**內部補登工具**值得；**不做**客人自助訂位/即時桌位/庫存。
- 狀態流程：待聯繫 → 已報名待訂金 → 已收訂 → 完成/取消；提供催訂金清單。
- 補登欄位：預約類型、活動日期、人數、姓名、聯絡方式、來源、訂金、收款方式、匯款末五碼、收款日期、備註、狀態。
- 預約類型：DIY/校外教學、生態導覽、婚禮/包場、團體聚餐、家庭聚餐。

**金流延後**，三條件全成立才做：(a) 補登流程順跑完一整個旺季；(b) 量大到人工核對末五碼真的痛；(c) 老闆/會計談定錢落哪、誰處理發票/退款/取消。屆時**優先走 SO-BUY「活動訂金商品」**（複用現有商家、免新申請金流），而非直接串閘道。

## 6. P7.5｜官網直接聯繫 + FB 引流 + 追蹤

### 客服／官網直接聯繫
- ⚠ **Facebook 嵌入式聊天外掛（Chat Plugin）Meta 已於 2024/5/9 停用**，不要在官網嵌 FB 聊天框（會是壞的 widget）。
- 改用**直接開對話的按鈕**（浮動聯繫鈕）：`m.me/ROSEBOXGOOD`（Messenger 深連結，穩定）、**LINE 官方帳號加好友連結**（台灣主力窗口）、電話。
- 加一個**「預約/洽詢申請表」→ 寫進 P8 私有預約 Sheet**，把訪客變成一筆進帳。
- 不自建真人即時客服（無人力守）。

### FB 引流
- **FB → 官網**：粉專 CTA 按鈕設「立即預訂/了解更多」連官網；貼文/簡介放連結；FB/IG 廣告落地頁設成本體驗頁（本頁本就是為承接廣告落地設計）；IG link in bio。
- **官網 → FB**：上面的 m.me／LINE 按鈕，把訪客導進既有對話下單流程。

### 追蹤（可提前到 MVP）
- 連結加 **UTM 參數**、裝 **Meta Pixel + GA4**，否則無法得知 FB 帶進多少、廣告值不值。
- 長期核心：**FB 或官網來的最終都進同一本預約帳**（呼應 P8）。

## 8. MVP vs Full 摘要

**最小可上線（MVP）**：接通 P3、後台預設常用內容、SEO 欄位人工鎖定＋checklist、可見頁價格單一來源；網域先用 github.io 亦可；預約走內部補登＋私有 Sheet＋純匯款；建議一併裝追蹤（Pixel/GA4）。
主要風險：誤改 SEO 鎖定欄位、人員採用度、文字重繪疑點。

**未來完整（Full）**：發布時靜態重生（GitHub Action 或 Apps Script→GitHub，後者需評估 repo 寫入安全）、全站單一來源、花曆上雲；自訂網域＋全套 SEO/連結同步；官網聯繫鈕＋FB 引流；預約客人網頁表單餵同一流程、SO-BUY 訂金整合、報表/催訂金自動化；金流 SO-BUY 先行。
主要風險：自動化寫 GitHub 的安全面、金流/個資/發票合規、照現況量做過頭。
